SDN Testbed Policies, Limits, Requirements

  • Privileged mode is not allowed. Kubernetes flag: allowPrivilegeEscalation
  • Host IPC namespace use is not allowed. Kubernetes flag: hostIPC
  • Host PID namespace use is not allowed. Kubernetes flag: hostPID
  • Use of hostNetwork and hostPort not allowed. Kubernetes flag: hostNetwork, hostPort. Please use Port Expose and HAProxy
  • The label pod.kubernetes.io/lifetime is required. Value must be integer.
  • Use of default namespace is not allowed. Always use your own username or assigned group namespace
  • CPU, Memory limits and requests must be always defined. See more details below of min/max requests
  • Max allocated (Requests) resources per namespace allowed for CPU:16 cores and Mem:32Gi
  • Max allocated (Limits) resources per namespace allowed for CPU:32 cores and Mem:64Gi
  • Max Persistent Volume Claims: 5 and total allowed storage per namespace: 50Gi
  • Pod default requests for CPU: 500m, Mem:500Mi, Storage:None
  • Pod minimum requests for CPU: 100m, Mem:100Mi, Storage:1G
  • Pod maximum requests for CPU: 4, Mem:5Gi, Storage:10G
  • hostpath restricted to only certain paths. The policy ensures the only directory that can be mounted as a hostPath volume is /storage/af/user/{USERNAME}. This is the same as your home directory on login-[1,2,3].hep.caltech.edu
  • Any capabilities added beyond the allowed list (AUDIT_WRITE, CHOWN, DAC_OVERRIDE, FOWNER, FSETID, KILL, MKNOD, NET_BIND_SERVICE, SETFCAP, SETGID, SETPCAP, SETUID, SYS_CHROOT) are disallowed.
